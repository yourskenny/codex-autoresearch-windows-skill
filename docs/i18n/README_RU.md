Идея: скажите Codex, что хотите улучшить, и уходите. Он изменит код, проверит результат, сохранит или откатит, и повторит. Вы вернётесь к журналу экспериментов и улучшенной кодовой базе.

Вдохновлено [autoresearch Карпати](https://github.com/karpathy/autoresearch), обобщено за пределы ML на всё, что можно проверить механически: покрытие тестами, ошибки типов, задержки, предупреждения lint, уязвимости безопасности, готовность к релизу — если команда может определить, стало ли лучше, цикл может итерировать.

## Быстрый старт

```text
# Установка в Codex (рекомендуется)
$skill-installer install https://github.com/leo-lilinxiao/codex-autoresearch
```

Перезапустите Codex, откройте ваш проект и начинайте:

```
Вы:    $codex-autoresearch
       Хочу избавиться от всех типов `any` в моём TypeScript-коде

Codex: Нашёл 47 вхождений `any` в src/**/*.ts.
       Каталог Results: ./autoresearch-results/
       Метрика: количество `any` (текущее: 47), направление: уменьшение
       Проверка: подсчёт grep + tsc --noEmit как guard
       Режим запуска: foreground или background?

Вы:    Background, go. Пусть работает всю ночь.

Codex: Запускаю фоновое выполнение — базовая линия: 47. Итерирую.
```

Каждое улучшение накапливается. Каждая неудача откатывается. Всё записывается в журнал.

Ручное копирование, symlink и пользовательская установка описаны в [INSTALL.md](../INSTALL.md). Полное руководство в [GUIDE.md](../GUIDE.md).

## Как это работает

```
Вы говорите фразу  →  Codex сканирует и подтверждает  →  Вы говорите "go"
                                                              |
                                               +--------------+--------------+
                                               |                             |
                                          foreground                    background
                                        (текущая сессия)           (отдельно, на ночь)
                                               |                             |
                                               +--------------+--------------+
                                                              |
                                                              v
                                                    +-------------------+
                                                    |     Цикл         |
                                                    |                   |
                                                    |  изменить одно   |
                                                    |  git commit       |
                                                    |  запустить verify |
                                                    |  лучше? оставить |
                                                    |  хуже? revert    |
                                                    |  записать результ.|
                                                    |  повторить       |
                                                    +-------------------+
```

Вот и всё. Вы выбираете одно из двух: foreground оставляет цикл в текущей сессии, background передаёт его отдельному процессу, чтобы вы могли спать. Один и тот же цикл, но они не работают одновременно.

## Что вы говорите vs что происходит

| Что вы говорите | Что происходит |
|----------------|----------------|
| «Улучши покрытие тестами» | Итерирует до достижения цели или прерывания |
| «Исправь 12 падающих тестов» | Чинит по одному, пока не останется ни одного |
| «Почему API возвращает 503?» | Ищет корневую причину с фальсифицируемыми гипотезами |
| «Этот код безопасен?» | Аудит STRIDE + OWASP, каждая находка подкреплена кодом |
| «Готово к релизу» | Проверяет готовность, генерирует чеклист, контролирует выпуск |
| «Хочу оптимизировать, но не знаю что» | Анализирует репозиторий, предлагает метрики, генерирует конфигурацию |

За кулисами Codex сопоставляет вашу фразу с одним из 7 режимов (loop, plan, debug, fix, security, ship, exec). Вам не нужно выбирать.

## Что Codex определяет автоматически

Вы не пишете конфигурацию. Codex выводит всё из вашей фразы и вашего репозитория:

| Что нужно | Как получает | Пример |
|-----------|-------------|--------|
| Цель | Ваша фраза | «избавиться от всех типов any» |
| Область | Сканирует структуру репозитория | `src/**/*.ts` |
| Метрика | Предлагает на основе цели + инструментов | количество any (текущее: 47) |
| Направление | Выводит из «улучшить» / «уменьшить» / «устранить» | уменьшение |
| Проверка | Сопоставляет с инструментами репозитория | подсчёт `grep` + `tsc --noEmit` |
| Guard | Предлагает при наличии риска регрессии | `npm test` |

Перед запуском Codex всегда показывает, что нашёл, и просит подтверждения. Затем вы выбираете foreground или background и говорите «go».
По умолчанию каталог Results остаётся в контексте запуска: если вы запустили Codex внутри git-репозитория, корень этого репозитория становится workspace root по умолчанию; если вы запустили его вне git-репозитория, workspace root по умолчанию становится текущий каталог запуска. Codex не должен молча расширять это до родительского каталога, если вы явно не подтвердили более широкий multi-repo workspace. В сводке подтверждения перед запуском всегда должен быть показан выбранный каталог Results.

## Когда заходит в тупик

Вместо слепых повторов цикл эскалирует:

| Триггер | Действие |
|---------|----------|
| 3 неудачи подряд | **REFINE** — корректировка в рамках текущей стратегии |
| 5 неудач подряд | **PIVOT** — попробовать принципиально другой подход |
| 2 PIVOT без прогресса | **Веб-поиск** — искать внешние решения |
| 3 PIVOT без прогресса | **Остановка** — сообщить, что нужно вмешательство человека |

Один успех сбрасывает все счётчики.

## Журнал результатов

Каждая итерация записывается в `autoresearch-results/results.tsv`:

```
iteration  commit   metric  delta   status    description
0          a1b2c3d  47      0       baseline  initial any count
1          b2c3d4e  41      -6      keep      replace any in auth module
2          -        49      +8      discard   generic wrapper introduced new anys
3          d4e5f6g  38      -3      keep      type-narrow API response handlers
```

Неудачные эксперименты откатываются в git, но остаются в журнале. Журнал — это настоящий аудиторский след, а `autoresearch-results/state.json` — снимок для возобновления.

## Дополнительные возможности

Подробности в [GUIDE.md](../GUIDE.md):

- **Обучение между запусками** — уроки прошлых запусков влияют на генерацию будущих гипотез
- **Параллельные эксперименты** — одновременная проверка до 3 гипотез через git worktrees
- **Возобновление сессии** — прерванные запуски продолжаются с последнего согласованного состояния
- **Режим CI/CD** (`exec`) — неинтерактивный, вывод в JSON, для конвейеров автоматизации
- **Двойная проверка** — раздельные verify (стало лучше?) и guard (ничего не сломалось?)
- **Session hooks** — устанавливаются автоматически; поддерживают состояние Codex между сессиями

## FAQ

**Делает только маленькие изменения. Может ли пробовать более крупные идеи?**
По умолчанию цикл предпочитает маленькие проверяемые шаги — это сделано намеренно. Но он может мыслить масштабнее: опишите более крупную гипотезу в промпте (например, «замени механизм attention на linear attention и запусти полную оценку»), и он проверит это как один эксперимент. Оптимальный подход: человек задаёт направление исследования, агент берёт на себя интенсивное выполнение и анализ.

**Это больше для инженерной оптимизации или для исследований?**
Сильнее всего работает, когда цель и метрика ясны — поднять покрытие, уменьшить ошибки, снизить задержку. Если само направление исследования неопределённо, сначала используйте режим `plan` для разведки, затем переключитесь на `loop`, когда поймёте, что измерять. Думайте об этом как о сотрудничестве человека и ИИ: вы даёте суждение, агент даёт скорость итерации.

**Как остановить?** Foreground: прервите Codex. Background: `$codex-autoresearch`, затем попросите остановить.

**Может возобновить после прерывания?** Да. Автоматически возобновляется из `autoresearch-results/state.json`.

**Как использовать в CI?** `Mode: exec` с `codex exec`. Вся конфигурация заранее, вывод JSON, коды выхода 0/1/2.
